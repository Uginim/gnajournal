# Claude Code 프로젝트 가이드

이 프로젝트에서 Claude Code가 따라야 할 규칙들입니다.

---

## 학습 노트 정리 규칙

### 목적

사용자가 질문한 개념들을 `docs/notes/` 폴더에 체계적으로 정리하여 학습 자료로 활용한다.

### 폴더 구조

```
docs/notes/
├── README.md          # 전체 목차
├── java/              # Java/JVM 관련
├── kotlin/            # Kotlin 언어
├── ktor/              # Ktor 프레임워크
├── database/          # DB, SQL, Exposed ORM
├── web/               # HTML, CSS, HTTP, 웹 개념
└── devops/            # Docker, 배포, CI/CD
```

### 언제 정리하는가?

다음 상황에서 `docs/notes/`에 문서를 생성한다:

1. **개념 질문**: "~이 뭐야?", "~가 뭔지 설명해줘"
2. **비교 질문**: "A vs B 차이가 뭐야?"
3. **설정/해결 과정**: 에러 해결, 환경 설정 등 반복될 수 있는 작업
4. **사용자 요청**: "이것도 정리해줘"

### 정리하지 않는 경우

- 단순 코드 작성 요청
- 일회성 디버깅
- 프로젝트 특정 로직 (일반적이지 않은 내용)

### 파일 작성 규칙

1. **파일명**: kebab-case 사용 (예: `class-file-version.md`)
2. **언어**: 한국어로 작성
3. **구조**:
   ```markdown
   # 제목

   ## 개념
   간단한 설명

   ## 상세 내용
   표, 예시 코드 등

   ## 참고
   관련 링크나 추가 정보
   ```

4. **README.md 업데이트**: 새 문서 추가 시 목차에도 추가

### 분류 기준

| 폴더 | 포함 내용 |
|-----|----------|
| `java/` | JVM, JDK, 클래스 파일, GC, 메모리 등 |
| `kotlin/` | 코틀린 문법, 코루틴, 확장 함수 등 |
| `ktor/` | 라우팅, 플러그인, 인증, HTML DSL 등 |
| `database/` | SQL, Exposed ORM, 트랜잭션, 마이그레이션 |
| `web/` | HTTP, REST, HTML, CSS, HTMX |
| `devops/` | Docker, CI/CD, 배포, 모니터링 |

## 작업 일지 규칙

### 목적

프로젝트 진행 상황을 `docs/worklog/`에 날짜별로 기록하여 히스토리를 관리한다.

### 폴더 구조

```
docs/worklog/
├── README.md          # 전체 목차 (날짜별 링크)
├── 2024-11-26.md      # 날짜별 작업 일지
├── 2024-11-27.md
└── ...
```

### 언제 작성하는가?

1. **세션 시작 시**: 새 날짜면 새 파일 생성
2. **주요 작업 완료 시**: 완료 항목 업데이트
3. **커밋 전**: 오늘 작업 내용 정리
4. **이슈 해결 시**: 이슈 및 해결 섹션에 기록

### 파일 작성 규칙

1. **파일명**: `YYYY-MM-DD.md` 형식
2. **구조**:
   ```markdown
   # YYYY-MM-DD 작업 일지

   ## 오늘의 목표
   - 목표 1
   - 목표 2

   ---

   ## 완료 항목
   ### 1. 작업 제목
   - 상세 내용

   ---

   ## 이슈 및 해결
   ### 이슈 제목
   - **문제**:
   - **원인**:
   - **해결**:

   ---

   ## 다음 작업
   - [ ] 할 일 1
   - [ ] 할 일 2

   ---

   ## 메모
   - 기타 메모 사항
   ```

3. **README.md 업데이트**: 새 날짜 파일 추가 시 목차에도 추가

## 기타 규칙

- 코드 수정 시 기존 스타일 따르기
- 커밋 메시지는 한국어로 작성 가능
